36openMTPK Linear Algebra Module
37==============================
38since the Matrix class is a template class, there are classes created
39for both Doubles and Integers
40
41The template Matrix class and mtx struct were converted to type double
42methods instead of declaring a method for each type.
43 *NOTE* there is a respective 'INTEGER' version of each method that
44 defaulted to type double, my recommendation is to use the default
45 methods that converted to type double as data will more than likely
46 follow the scheme.
47
48 SCHEMES:
49 class Matrix (int) -> class MatrixI
50 an INTEGER type mtx struct is not possible given floating
51 point requirements for certain cases of cassert
52
53FEATURES :
54 CLASS : Matrix
55 - The default Matrix Class defaulting to floating type
56 - MatrixI Class features same methods but Integer types
57 METHODS :
58 mult
59 scalar_mult
60 mult_elem
61 sqr_err
62 add
63 scalar_add
64 sub
65 transpose
66 sum
67 mean
68 concatenate
69 diag
70 print_shape
71 print_mtx
72 fill_index
73
74 CLASS : mtx
75 - Closely coupled with the Matrix Class
76 - mtxI Class features same methods but Integer types
77 METHODS :
78 zeros
79 ones
80 randn
81 rand
82
83 TODO :
84 - Vectors Class? Matrix Class is capable of creating Vectors
85 but possible error prone to not fixing to a dimension size.
86
87 - Tensor Class?
88
89 USAGE :
90 >> import openmtpk
91 >> # create matrix of 3,4 size
92 >> mtx = openmtpk.linalg.Matrix(3, 4)
93 OR
94 >> from openmtpk import linalg
95"""
